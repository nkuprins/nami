## Architecture

**Stack:** Spring Boot 4.0.6, Java 25, PostgreSQL 17. Gradle 9.4.1 with Kotlin DSL. Lombok for boilerplate.

### Package layout

```
com.app.backend
├── config/         # AppProperties (record), CorsConfig, S3Config, SecurityConfig
├── controller/     # AuthController, PropertyController, SavedController, UploadController, GlobalExceptionHandler
├── dto/            # Request/response records; auth/ sub-package for auth DTOs
├── entity/         # JPA entities: User, Property, PropertyPhoto, PropertyPlan, PropertyPhone, PropertyTranslation, SavedProperty, tokens
├── enums/          # ListingType, PropertyCategory, PropertyStatus, PropertyCompletion, PropertyFeature, SupportedLocale
├── exception/      # AuthException (and subtypes) thrown by services, caught by GlobalExceptionHandler
├── job/            # Scheduled jobs: PropertyExpiryJob, InactiveAccountPurgeJob, TokenCleanupJob
├── mapper/         # PropertyMapper (entity → DTO)
├── repository/     # Spring Data JPA repositories
├── security/       # JwtService, JwtAuthFilter, CookieFactory, RateLimitFilter (Bucket4j)
├── service/        # AuthService, PropertyService, SavedService, UploadService, EmailService
└── spec/           # PropertySpec (JPA Criteria / Specification), PropertySearchCriteria
```

### Auth flow

Stateless JWT. `login` / `register` / `refresh` issue an **access token** (15 min, returned in JSON body) and a **refresh token** (7 days, HttpOnly cookie). `JwtAuthFilter` validates the bearer token and sets `UUID` as the `AuthenticationPrincipal`. Refresh tokens are stored hashed (SHA-256) in the DB. Email verification and password reset use opaque tokens stored hashed.

Rate limiting via Bucket4j (`RateLimitFilter`) — keyed by IP on auth endpoints.

### Property lifecycle

Properties have a status (`ACTIVE` / `INACTIVE`) and an `expires_at` timestamp. `PropertyExpiryJob` runs daily at 2 AM:
1. Sends expiry warning emails 7 days before expiry (digest per owner).
2. Flips `ACTIVE → INACTIVE` on expired listings.
3. Permanently deletes `INACTIVE` listings expired >90 days ago (including S3 media).

Owners can renew a listing via `POST /api/properties/{id}/renew`.

### Multilingual content

`PropertyTranslation` holds locale-keyed (`lv` / `en` / `ru`) title + description per property. Mapped as `Map<String, PropertyTranslation>` on `Property`. Email jobs resolve the title with lv → en → ru fallback.

### Storage

S3 presigned PUT URLs are generated by `UploadService` for client-side direct upload. Media is served via CloudFront CDN. `UploadService.deleteObjects` is called after-commit from jobs when purging media.

### Database schema (`db/schema.sql`)

PostgreSQL with:
- Extensions: `pgcrypto` (UUID generation, bcrypt), `cube` + `earthdistance` (geospatial proximity queries)
- Tables: `users`, `properties`, `property_features` (junction), `property_photos`, `property_plans`, `property_phones`, `property_translations`, `saved_properties`, token tables
- `properties` uses `district_slug` / `city_slug` — slug validity enforced at API layer; canonical mapping lives in frontend `locations.ts`
- `completion` field only valid for `listing_type = 'NEW_PROJECT'`; apartments cannot have `land_m2` — both enforced via CHECK constraints
- Geospatial index on `ll_to_earth(lat, lng)` for proximity queries
- `updated_at` auto-updated via trigger on `users` and `properties`

**Seed data** (`db/seed.sql`) — 5 dev users (password `Secret123!`) and sample properties. Apply manually.

## Configuration

All sensitive values are env-variable driven with dev defaults. Key variables:

| Variable | Default | Purpose |
|---|---|---|
| `DB_URL` | `jdbc:postgresql://localhost:5432/baltnami` | PostgreSQL connection |
| `DB_USERNAME` / `DB_PASSWORD` | `postgres` | DB credentials |
| `JWT_SECRET` | `dev-secret-...` | HMAC signing key (≥32 chars) |
| `CORS_ALLOWED_ORIGINS` | `http://localhost:5173` | Frontend origin |
| `S3_BUCKET` | `baltnami-uploads` | S3 bucket name |
| `AWS_REGION` | `eu-north-1` | AWS region |
| `CDN_URL` | CloudFront URL | Media base URL |
| `RESEND_API_KEY` | _(empty)_ | Transactional email |
| `FRONTEND_URL` | `http://localhost:5173` | Used in email links |
| `COOKIE_SECURE` | `false` | Set `true` in prod |

Config is bound via `AppProperties` record (`app.*` prefix).

## Testing

Integration tests use **Testcontainers** (PostgreSQL 17) — Docker must be running. `IntegrationTestBase` spins up a shared container, applies `db/schema.sql`, and wires MockMvc with Spring Security. S3Client and S3Presigner are mocked. Tests are `@Transactional` (auto-rollback). Rate limit buckets are evicted before each test.

Unit tests use Mockito (loaded as javaagent) and **Instancio** for test data generation. **ArchUnit** enforces package dependency rules.

JaCoCo enforces 70% line coverage minimum — `./gradlew check` will fail below threshold.
